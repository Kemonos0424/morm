# MORM（モルム）プロジェクト仕様書

## 1. プロダクトビジョンとサービス概要
* [cite_start]**プロダクト名**: MORM（モルム） [cite: 167, 366]
* [cite_start]**名前の由来**: Memory + Swarm（群れ） / Momentum。分散型ノードの有機的な繋がりと勢いを表現 [cite: 168, 170, 194]
* [cite_start]**スローガン**: "The Swarm for Every Frame"（すべてのフレームに、群れの力を） [cite: 197, 399]
* [cite_start]**ビジョン**: 中央集権型サーバーに依存せず、世界中のデバイスを繋ぐ完全分散型・超高速動画プラットフォーム兼P2Pコマース基盤 [cite: 400]
* [cite_start]**コア体験**: TikTokライクな縦型動画の無限スクロールと、物理的証明に基づく詐欺不可能なP2P取引 [cite: 4, 368]

### ターゲットユーザーと提供価値
* [cite_start]**一般視聴者**: 広告・通信制限のストレスがない動画体験。デバイスの空きリソースを用いた中継によるトークン報酬 [cite: 403]
* [cite_start]**クリエイター**: AI検閲のない自由な表現環境。初期値絶対主義によるオリジナル作品の保護 [cite: 403]
* [cite_start]**購入者・販売者**: 梱包・開封の動画エビデンスによる詐欺リスクゼロのP2Pダイレクト売買 [cite: 403]
* [cite_start]**ノード提供者 (NodePower)**: GPUや空き容量の提供による、AI検証やエンコードに対する高額報酬 [cite: 403]

## 2. システム・アーキテクチャ (MORM Core)
* **P2Pメッシュネットワーク階層**: 
  * [cite_start]**Validator Node (NodePower)**: GPU/CPUフル稼働。AIエンコード、動体解析、ブロック生成、スクリーニング検証 [cite: 102, 372]
  * [cite_start]**Edge Node**: SSD・帯域提供。動画セグメント（MORM Cells）の保持と高速配信 [cite: 103, 372]
  * [cite_start]**Light Client**: スマホ・PC・ゲーム機等。動画視聴とUI描画、一時的なキャッシュ中継 [cite: 104, 372]
* [cite_start]**通信プロトコル**: 超高速ストリーミングを実現するQUICプロトコルを採用 [cite: 77, 78]
* [cite_start]**独自ブロックチェーン (MORM Chain)**: 全ノード分散型、DAGベースの並列処理。PoUW（Proof of Useful Work）による実用的計算証明 [cite: 44, 45, 46, 106, 202]
* [cite_start]**ウォレットレスID**: パスキー（生体認証）とデバイス・バインドを活用し、シードフレーズなしでシームレスにログイン可能 [cite: 74, 75, 378]

## 3. 超高速動画ストリーミング
* [cite_start]**MORM Cells**: 動画を30fps以下・WebM形式で「3秒」ごとのセグメントに分割 [cite: 18, 204, 374]
* **50% / 10% サイクル**:
  * [cite_start]10%（0.3秒）再生で次のCell取得をバックグラウンド予約 [cite: 294, 375]
  * [cite_start]50%（1.5秒）再生で古いCellをメモリからパージし、デバイスの負荷を軽減 [cite: 295, 375]
* [cite_start]**ロード中のUX**: ロード時間にAIアノテーションや予測市場のミニゲームを提供し、待ち時間をタスク化して体感ゼロへ [cite: 29, 30, 89, 90]
* [cite_start]**自動クリーニング**: 人気のない動画や再生済みキャッシュを自動消去し、ストレージ効率と多様性を維持 [cite: 122, 123, 138, 140]

## 4. エコシステムとインセンティブ (Tokenomics)
* [cite_start]**透明な1%手数料モデル**: 決済通貨（BTC, SOL等）をMORMトークンへスワップ。運営手数料はスマートコントラクトで1.0%のみ自動徴収 [cite: 233, 234, 380]
* [cite_start]**報酬のマルチ評価**: GPU（AI解析・エンコード）は特大、SSD（データ保持）は中、帯域（P2P配信）は大の報酬を分配 [cite: 82]
* **投稿権限のステータス制 (Proof of Contribution)**: 
  * [cite_start]**Level 1 (Viewer)**: ノード稼働（小）、視聴、コメント [cite: 134]
  * [cite_start]**Level 2 (Creator)**: トークン保有＋NodePower提供で、1日3本の動画投稿が可能 [cite: 134]
  * [cite_start]**Level 3 (Publisher)**: 大量トークン保有＋GPU提供で無制限投稿・4K配信が可能 [cite: 134]

## 5. コンテンツ保護とスクリーニング
* [cite_start]**初期値絶対主義**: コピー動画や切り抜きを排除し、オリジナルを保護する [cite: 335, 382]
* [cite_start]**V-Hash (ビデオ・ハッシュ)**: 動画の特徴量や音声をハッシュ化・ベクトル照合し、類似動画のアップロードを即座に拒否・一斉消去 [cite: 312, 313, 340, 341, 349, 383]
* [cite_start]**Generation ID**: 生成AI作成動画にプロンプト等のシード値を持たせ、MORM Chain上に最初の1本（オリジナル）の証明として刻印 [cite: 343, 344, 345]

## 6. MORM Shop ＆ 信頼プロトコル
* [cite_start]**物理的証拠の証明 (Proof of Physical Evidence)**: MORM専用カメラで梱包・開封を撮影。最新ブロックハッシュを透かしとして埋め込み、改ざんを防止 [cite: 238, 239, 240, 255, 388]
* [cite_start]**スマート・エスクロー**: 資金の99%をロック。AI解析（動体解析など）やバリデーションで梱包・開封動画が一致すればロック解除 [cite: 235, 242, 267, 389]
* [cite_start]**Node-Lock & Slash (厳格な制裁)**: 不正・すり替え検知時、資金の組み戻し、ノード権限とトークンの没収、デバイス・生体認証レベルでの永久追放を実行 [cite: 243, 246, 247, 248, 273, 274, 390]

## 7. PoC (概念実証) の環境想定
* [cite_start]**Origin / Validator**: RTX 6000 BW サーバーを利用し、動画のCell分割、ハッシュ埋め込み、AI検証・動体解析を実行 [cite: 304, 393]
* [cite_start]**Edge / Client**: MacBook Pro M4等を利用し、10/50%サイクルの視聴テストや開封動画の疑似撮影・アップロードテストを実施 [cite: 304, 394]
* [cite_start]**Local Chain**: Docker上のPrivate Chainにて、1%手数料処理とエスクローのステータス管理を検証 [cite: 304]
