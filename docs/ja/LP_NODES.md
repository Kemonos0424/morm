## なぜMORMノードか

### 既存DePINとの比較

| プロジェクト | 提供リソース | 報酬の重み | MORMとの差 |
|---|---|---|---|
| Filecoin | ストレージ | 大 | MORMは動画+AIで多重収益 |
| Theta | 帯域 | 中 | MORMはエンコード+検証も報酬 |
| Render | GPU（3D） | 大 | MORMは動画特化、需要安定 |
| Livepeer | GPU（トランスコード） | 大 | MORMはAIも統合 |
| Akash | 汎用コンピュート | 中 | MORMは特化型ジョブで効率高 |
| **MORM** | **GPU / 帯域 / SSD / 稼働時間** | **多重最大化** | **動画+AI+配信+検証の統合** |

MORMは「**1つのマシンが4種類の仕事で稼ぐ**」設計。リソースの相互補完で、稼働率を最大化できます。

## セットアップ

### 必要なもの

- インターネット接続（最低 上り20Mbps）
- 対応OS: Linux、macOS、Windows、Docker
- 任意で: GPU（NVIDIA / AMD / Apple Silicon）、SSD空き容量

### Step 1: ノード・ソフトウェアをダウンロード

```
curl -fsSL https://get.morm.network | sh
```

または、対応するパッケージマネージャから:

- macOS: `brew install morm-node`
- Windows: 公式インストーラ（MSI）
- Linux: `apt install morm-node` / `yum install morm-node`

### Step 2: 初回起動と認証

```
morm-node init
```

スマホで生体認証 → デバイスがMORM Chainに登録 → ノード稼働開始。

### Step 3: リソース提供を選択

```
morm-node configure
  --gpu enable
  --bandwidth-cap 100mbps
  --storage-cap 100GB
  --active-hours 24
```

### Step 4: 報酬の確認

ダッシュボード（Web、CLI、モバイルアプリ）でリアルタイムに監視:
- 提供リソース量
- 完了したPoUW仕事
- 累積MORM報酬
- ステーキング状況

## 各種ハードウェアの最適化ガイド

### RTX 6000 Ada / RTX 6000 BW

最強構成。AI解析+トランスコード+配信を並列実行。

- 推奨: Linux、24時間稼働
- 月間想定: 8,000-25,000 MORM
- 電力消費: ~300W（電気代 ~$30-80/月地域差）

### RTX 4090 / 5090

ハイエンド・ゲーミングPC兼用。アイドル時のみ稼働も可能。

- 推奨: Windows / Linux
- 月間想定: 1,500-6,000 MORM
- ゲーム中は自動停止設定可

### Apple Silicon（Mac mini M2/M3/M4）

低消費電力・高効率。動画エンコードに特化（VideoToolbox）。

- 推奨: macOS、ファンレス常時稼働
- 月間想定: 500-2,000 MORM
- 電力消費: ~30W

### Raspberry Pi クラスター

ストレージ+帯域専用ノード。低初期投資で多台数展開可能。

- 推奨: Linux ARM
- 月間想定: 50-200 MORM / 台
- 専用ノード機器（MORM Hardware）として将来展開予定

## NodePower vs Edge Node の選択

| 種別 | 必要要件 | 月間報酬目安 |
|---|---|---|
| **Validator (NodePower)** | 高性能GPU + 24h稼働 + 安定回線 | 5,000-25,000 MORM |
| **Edge Node** | SSD + 帯域 + 安定稼働 | 500-3,000 MORM |
| **Light Client** | スマホ・PCのみ | 5-100 MORM |

最初はLight Clientから始めて、Edge Node、NodePowerと段階的に拡張するのが推奨ルートです。

---

## ステーキング（Bonding）の役割

ノードを稼働させるには、Tier別の最小ステーク（Bond）が必要です:

| Tier | 最小ステーク |
|---|---|
| Light Client | 0 MORM |
| Edge Node | 100 MORM |
| Validator | 10,000 MORM |
| Validator (Publisher Tier) | 100,000 MORM |

ステーク額が大きいほど、優先的に仕事が割り当てられます。**不正検知時はステークがSlashされるため、誠実な稼働が必須です。**

---

## 不正リスクと安全策

| リスク | MORMの対策 |
|---|---|
| 偽装ノード（リソース嘘の申告） | チャレンジレスポンス検証、Slash対象 |
| Sybil攻撃 | 生体認証ベースのID、永久ban |
| 51%攻撃 | PoUWの「有用仕事」要件、Validator多様性 |
| ハードウェア故障 | グレースピリオド、再開ペナルティ免除 |
| ネット切断 | 一時的な報酬減、復旧で完全復帰 |

**ノード提供者として誠実に稼働している限り、Slashリスクはほぼゼロです。**

## FAQ（ノード運営向け）

### Q. 初期投資はどれくらい必要？

A. ゼロから可能です。手持ちのスマホ・PCがそのままLight Clientとして稼働します。本格的にやりたい場合は、ゲーミングPC（10-20万円）から、専用GPUサーバー（100万円〜）まで自由に選べます。

### Q. 電気代は採算に合いますか？

A. 地域とハードウェアに依存します。日本の家庭電気料金（30円/kWh）でRTX 4090を24h稼働すると、月の電気代 約$60。月収目安1,500-6,000 MORMが現在のレートで$300-1200相当なら採算プラスです。市場価格と各国電気料金を考慮した収支シミュレータを提供予定です。

### Q. 複数台のノードを同じアカウントで運用できる？

A. はい。1つのMORM IDに複数デバイスを紐付けられます。ただしSybil攻撃を防ぐため、デバイスごとに生体認証またはハードウェアバインディングが必要です。

### Q. データセンターからの大規模運用は可能？

A. はい。MORM Phase 1以降、データセンター運営者向けの「Operator License」を提供予定です。法人契約・KYCを経て、1000台規模の運用が可能になります。

### Q. NVIDIA以外（AMD、Apple Silicon）は動きますか？

A. はい。MORM AI/エンコード処理はROCm（AMD）、VideoToolbox（Apple Silicon）にも対応。ただしCUDA最適化のNVIDIAが現時点で最も高効率です。
